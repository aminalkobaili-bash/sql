## 1. Создание индексов для внешних ключей

- Создать B-Tree индексы:
  - для всех столбцов, являющихся внешними ключами
- Формат имени индекса:

```bash
idx_<table>_<column>
```

- Пример:

```bash
idx_menu_pizzeria_id
```

---

## 2. Проверка использования индексов

- Сформировать SQL-запрос:
  - вывод названия пиццы и пиццерии
- Проверить план выполнения:

```bash
EXPLAIN ANALYZE
```

- Убедиться:
  - используется `Index Scan`
- При необходимости:
  - отключить последовательное сканирование для тестирования

---

## 3. Функциональные индексы

- Создать индекс:

```bash
idx_person_name
```

- Особенность:

  - индекс строится по значению `UPPER(name)`

- Подготовить запрос:
  - демонстрирующий использование индекса
- Проверка через:

```bash
EXPLAIN ANALYZE
```

---

## 4. Многоколоночные индексы

- Создать индекс:

```bash
idx_person_order_multi
```

- Столбцы:

  - `person_id`
  - `menu_id`
  - `order_date`

- Цель:

  - оптимизация запроса с фильтрацией по этим полям

- Ожидаемый результат:

```bash
Index Only Scan
```

---

## 5. Уникальные индексы

- Создать уникальный индекс:

```bash
idx_menu_unique
```

- Столбцы:

  - `pizzeria_id`
  - `pizza_name`

- Назначение:

  - исключение дубликатов

- Проверка через:

```bash
EXPLAIN ANALYZE
```

---

## 6. Частично уникальные индексы

- Создать частичный индекс:

```bash
idx_person_order_order_date
```

- Условия:

  - применяется только для даты `2022-01-01`

- Столбцы:

  - `person_id`
  - `menu_id`

- Ожидаемый результат:

```bash
Index Only Scan
```

---

## 7. Оптимизация производительности

- Проанализировать SQL-запрос с оконной функцией.
- Использовать:

```bash
EXPLAIN ANALYZE
```

- Создать индекс:

```bash
idx_1
```

- Цель:

  - уменьшение времени выполнения запроса

- Подход:

  - тестировать разные варианты индексов
  - удалять индекс перед повторной проверкой

- Подтвердить улучшение:
  - сравнить `Execution Time` до и после оптимизации
