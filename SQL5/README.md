## 1. Создание представлений

- Создать два представления:
  - для пользователей женского пола
  - для пользователей мужского пола
- Структура должна соответствовать таблице `person`.
- Пример имён:
  - `v_persons_female`
  - `v_persons_male`

---

## 2. Объединение данных из представлений

- Использовать ранее созданные представления.
- Получить единый список имён пользователей.
- Отсортировать результат по имени.

---

## 3. Генерация диапазона дат

- Создать представление:
  - `v_generated_dates`
- Использовать:

```bash
generate_series(...)
```

- Диапазон:
  - с `2022-01-01` по `2022-01-31`
- Тип данных: `DATE`
- Добавить сортировку.

---

## 4. Поиск пропущенных дат

- Использовать представление `v_generated_dates`.
- Определить даты:
  - отсутствующие в посещениях пользователей.
- Отсортировать по дате.

---

## 5. Операции над множествами

- Реализовать выражение:

```bash
(R − S) ∪ (S − R)
```

- Где:
  - R — посещения за `2022-01-02`
  - S — посещения за `2022-01-06`
- Работать со значениями `person_id`.
- Сохранить результат в представлении:
  - `v_symmetric_union`
- Добавить сортировку.

---

## 6. Расчёт скидки

- Создать представление:
  - `v_price_with_discount`
- Вывести:
  - имя пользователя
  - название пиццы
  - исходную цену
  - цену со скидкой (−10%)
- Привести значение скидки к типу `integer`.
- Отсортировать:
  - по имени
  - по названию пиццы

---

## 7. Материализованные представления

- Создать материализованное представление:
  - `mv_dmitriy_visits_and_eats`
- Условия:
  - пользователь: Dmitriy
  - дата: `2022-01-08`
  - цена < 800
- Данные должны быть сохранены при создании.

---

## 8. Обновление данных представления

- Добавить новую запись:
  - соответствующую условиям предыдущего пункта
  - с другой пиццерией
- Обновить материализованное представление.

---

## 9. Удаление объектов

- Удалить:
  - все созданные представления
  - материализованные представления
