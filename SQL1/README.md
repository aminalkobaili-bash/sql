## 1. Подготовка базы данных

- Использовать PostgreSQL (актуальная версия).
- Создать отдельную базу данных для выполнения заданий.
- Убедиться в наличии следующих таблиц:
  - `pizzeria`
  - `person`
  - `menu`
  - `person_visits`
  - `person_order`

---

## 2. Запросы выборки данных

### 2.1 Базовая выборка

- Получить список людей (имя и возраст), проживающих в городе `Казань`.

---

### 2.2 Фильтрация и сортировка

- Выбрать имена и возраст всех женщин из города `Казань`.
- Отсортировать результат по имени.

---

### 2.3 Диапазоны значений

- Сформировать два запроса:
  - с использованием операторов сравнения (`>=`, `<=`)
  - с использованием `BETWEEN`
- Условие:
  - рейтинг пиццерий от `3.5` до `5`
- Вывести:
  - название
  - рейтинг
- Отсортировать по рейтингу.

---

### 2.4 Уникальные значения и условия

- Получить уникальные `person_id`:
  - посетившие пиццерии с `2022-01-06` по `2022-01-09`
  - либо посещавшие пиццерию с `id = 2`
- Отсортировать по `person_id` по убыванию.

---

### 2.5 Формирование вычисляемых полей

- Создать вычисляемое поле `person_information` в формате:

```bash
Name (age:X,gender:'Y',address:'Z')
```

- Отсортировать результат по этому полю.

---

### 2.6 Подзапросы в SELECT

- Получить имена людей через подзапрос в `SELECT`.
- Условия:
  - `menu_id IN (13, 14, 18)` (реализовать без использования `IN`)
  - дата заказа: `2022-01-07`
- Использовать вложенный запрос в части `SELECT`.

---

### 2.7 Условная логика

- Добавить вычисляемое поле `check_name`.
- Логика:

```bash
если имя = 'Denis' → true
иначе → false
```

- Использовать конструкцию `CASE`.

---

### 2.8 Работа с диапазонами возраста

- Вывести:
  - id
  - имя
  - интервал возраста (`interval_info`)
- Логика:
  - 10–20 → interval #1
  - 21–23 → interval #2
  - иначе → interval #3
- Отсортировать по `interval_info`.

---

### 2.9 Фильтрация по условиям

- Выбрать все записи из `person_order`, где:
  - `id` — чётное число
- Отсортировать по `id`.

---

### 2.10 Подзапросы в FROM

- Использовать вложенный запрос в `FROM` на основе `person_visits`.
- Условия:
  - даты: `2022-01-07` — `2022-01-09`
- Вывести:
  - имя человека
  - название пиццерии
- Реализовать получение значений через подзапросы.
- Сортировка:
  - имя — по возрастанию
  - пиццерия — по убыванию
