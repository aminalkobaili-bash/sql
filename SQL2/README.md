## 1. Операции с множествами

### 1.1 Объединение данных

- Объединить данные из таблиц:
  - `menu` (id, название пиццы)
  - `person` (id, имя)
- Итоговые поля:
  - `object_id`
  - `object_name`

---

### 1.2 Объединение с подзапросами

- Модифицировать предыдущий запрос:
  - оставить только `object_name`
  - сохранить дубликаты
  - сначала вывести данные из `person`, затем из `menu`
- Использовать сортировку.

---

### 1.3 Уникальные значения без DISTINCT

- Получить список уникальных названий пицц.
- Ограничения:
  - не использовать `DISTINCT`, `GROUP BY`, `HAVING`, `JOIN`
- Отсортировать по убыванию.

---

### 1.4 Пересечение данных

- Найти записи, где совпадают:
  - `person_id`
  - дата (посещение и заказ в один день)
- Использовать таблицы:
  - `person_order`
  - `person_visits`
- Вывести:
  - дату действия
  - идентификатор человека
- Сортировка:
  - дата — по возрастанию
  - id — по убыванию

---

### 1.5 Разность множеств

- Найти различия между:
  - `person_order`
  - `person_visits`
- Условия:
  - дата `2022-01-07`
- Сохранить дубликаты результатов.

---

## 2. Работа с соединениями

### 2.1 Декартово произведение

- Получить все комбинации строк из:
  - `person`
  - `pizzeria`
- В результате:
  - сначала поля из `person`
  - затем поля из `pizzeria`

---

### 2.2 JOIN с форматированием

- Соединить:
  - `person_order`
  - `person`
- Вывести:
  - дату заказа
  - строку вида:

```bash
Name (age:X)
```

- Отсортировать по обоим столбцам.

---

### 2.3 NATURAL JOIN

- Переписать предыдущий запрос с использованием:

```bash
NATURAL JOIN
```

- Результат должен совпадать.

---

### 2.4 IN и EXISTS

- Реализовать два запроса:
  - с использованием `IN`
  - с использованием `EXISTS`
- Найти пиццерии, которые не посещались пользователями.

---

### 2.5 Комплексное объединение данных

- Получить:
  - имя пользователя
  - название пиццы
  - название пиццерии
- Использовать таблицы:
  - `person`
  - `menu`
  - `pizzeria`
  - `person_order`
- Сортировка:
  - имя
  - пицца
  - пиццерия (все по возрастанию)
