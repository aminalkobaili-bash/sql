## 1. Создание таблицы скидок

- Создать таблицу:

```bash
person_discounts
```

- Определить поля:

  - `id` — первичный ключ (тип аналогичен другим таблицам)
  - `person_id` — внешний ключ на таблицу `person`
  - `pizzeria_id` — внешний ключ на таблицу `pizzeria`
  - `discount` — значение скидки (тип `numeric`)

- Для внешних ключей задать имена ограничений:

```bash
fk_person_discounts_person_id
fk_person_discounts_pizzeria_id
```

## 2. Заполнение таблицы скидок

- Заполнить таблицу на основе данных `person_order`.
- Использовать:

```bash
INSERT INTO ... SELECT ...
```

- Логика:

  - группировка по `person_id`, `pizzeria_id`
  - вычисление скидки:
    - 1 заказ → 10.5
    - 2 заказа → 22
    - более → 30

- Генерация `id`:

```bash
ROW_NUMBER() OVER ()
```

## 4. Оптимизация и уникальность данных

- Создать индекс:

```bash
idx_person_discounts_unique
```

- Тип:

  - уникальный
  - многоколоночный

- Столбцы:

  - `person_id`
  - `pizzeria_id`

- Проверить использование:

```bash
EXPLAIN ANALYZE
```

---

## 5. Ограничения целостности

- Добавить ограничения:

```bash
NOT NULL
```

- Для полей:

  - `person_id`
  - `pizzeria_id`
  - `discount`

- Задать значения по умолчанию:

```bash
DEFAULT 0
```

- Ограничение диапазона:

```bash
0 <= discount <= 100
```

- Использовать имена:

```bash
ch_nn_person_id
ch_nn_pizzeria_id
ch_nn_discount
ch_range_discount
```

## 7. Автоматизация генерации ключей

- Создать последовательность:

```bash
seq_person_discounts
```

- Начальное значение:

```bash
1
```

- Настроить автоинкремент:

```bash
DEFAULT nextval(...)
```

- Обновить текущее значение последовательности:

  - равное количеству записей + 1

- Проверить:
  - отсутствие конфликтов по первичному ключу
