## 1. Базовая транзакция

- Использовать два параллельных сеанса работы с базой данных.
- В первом сеансе:
  - выполнить изменение значения (например, обновление рейтинга).
  - убедиться, что изменения видны внутри текущей транзакции.
- Во втором сеансе:
  - проверить, что изменения недоступны до фиксации.
- После выполнения:

```bash
COMMIT
```

- Убедиться, что изменения становятся доступными во втором сеансе.

## 2. Аномалия Lost Update

- Проверить текущий уровень изоляции:

```bash
SHOW TRANSACTION ISOLATION LEVEL;
```

- Установить уровень:

```bash
READ COMMITTED
```

- Использовать два сеанса:
  - в первом — изменить значение
  - во втором — выполнить другое изменение той же записи
- Проанализировать итоговое значение после фиксации изменений.


  - два параллельных изменения одной записи

## 4. Неповторяющееся чтение

- Установить уровень:

```bash
READ COMMITTED
```

- В первом сеансе:
  - выполнить чтение данных
- Во втором:
  - изменить эти данные
- В первом:
  - повторно выполнить чтение
- Зафиксировать изменение результата.

## 6. Фантомные чтения

- Установить уровень:

```bash
READ COMMITTED
```

- В первом сеансе:
  - выполнить агрегирующий запрос (например, суммирование)
- Во втором:
  - добавить новую запись:

```bash
INSERT
```

- В первом:
  - повторно выполнить запрос
- Зафиксировать изменение результата.

```bash
REPEATABLE READ
```

- Сравнить результаты.

## 8. Взаимоблокировка (deadlock)

- Использовать два параллельных сеанса.
- Выполнить операции обновления данных:
  - в разном порядке над одними и теми же строками
- Добиться ситуации:
  - взаимной блокировки транзакций
- Проанализировать:
  - поведение СУБД
  - завершение одной из транзакций системой
